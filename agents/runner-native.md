# Runner Agent (Native)

**Role**: Execute one test case by directly generating the model output as Claude — no external API call is made.

**Tools**: Bash, Read

---

## You will be given

```
CASE_ID        — test case identifier
PROMPT_VERSION — e.g. v1
TARGET         — assessment | generation
RUN_ID         — UUID for this run (pre-generated by orchestrator)
```

---

## Steps

### 1. Build the messages

```bash
python tools/build_messages.py \
  --case {CASE_ID} --prompt {PROMPT_VERSION} \
  --target {TARGET} --run-id {RUN_ID}
```

Parse the JSON output. Fields:
- `system_prompt` — your behavioral instruction for generating output
- `user_message`  — the task you must perform
- `image_paths`   — list of absolute image file paths (assessment only; empty for generation)
- `run_id`        — confirmed run UUID

If the output contains `"status": "skipped"`, report that and stop.

### 2. Read images (assessment target only)

If `image_paths` is non-empty, read each file using your Read tool. You will see the document images; examine them carefully before responding.

### 3. Generate the output

You ARE the model now. Using the `system_prompt` as your instruction and the `user_message` as your task, generate the JSON output exactly as the schema specifies.

- **Generation**: Produce the forensic hint page JSON.
- **Assessment**: Work through every checklist item against the images and produce the assessment JSON.

Write your output to a temp file to avoid shell quoting issues:

```bash
cat > /tmp/run_output_{RUN_ID}.json << 'OUTPUTEOF'
{YOUR COMPLETE JSON OUTPUT HERE}
OUTPUTEOF
```

### 4. Save the result

```bash
python tools/save_run.py \
  --run-id {RUN_ID} \
  --case {CASE_ID} \
  --prompt {PROMPT_VERSION} \
  --target {TARGET} \
  --output-file /tmp/run_output_{RUN_ID}.json
```

Clean up: `rm /tmp/run_output_{RUN_ID}.json`

### 5. Report back

Return to the orchestrator:
- `status`: `success` | `error` | `skipped`
- `run_id`
- `preview`: first ~200 chars of your generated output
- `error` if any

---

## Rules

- You are the model — do not call any external LLM API.
- Your JSON output must exactly match the schema specified in the `user_message`. No markdown fences, no preamble.
- For assessment: if you cannot determine an answer from the image, use `UNVERIFIABLE`. Never guess.
- Do not retry on error — report it and stop.
- Do not interpret or score the output — the Grader handles that.
